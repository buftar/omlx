## Tasks

### Documentation Structure (OBS-04)

Create docs/compression/ with:

1. **README.md** - Architecture overview
   - KV cache compression pipeline overview (AM + kvtc)
   - When to use compression
   - Performance expectations

2. **CONFIGURATION.md** - CompressionConfig reference
   - All configuration options
   - Default values
   - Runtime vs startup configuration

3. **CALIBRATION.md** - Calibration workflow guide
   - Running `omlx calibrate-kv <model>`
   - Calibration bundle format
   - When to re-calibrate

4. **TROUBLESHOOTING.md** - Common failure modes
   - Missing bundle error
   - Float16 input handling
   - Latency regression detection

### Benchmark Metrics Extension (Integration)

Extend benchmark report schema:

```python
# omlx/compression/benchmark.py
@dataclass
class BenchmarkReport:
    # ... existing fields ...
    compression_metrics: CompressionMetrics | None = None

@dataclass
class CompressionMetrics:
    compression_ratio: float
    decompression_latency_ms_per_layer: float
    cache_hit_rate: float
    cache_miss_rate: float
```

Update BenchmarkRunner to populate compression_metrics from ServerMetrics.

### Admin UI Dashboard (OBS-03 Extension)

Add compression section to admin dashboard:

1. **Compression Settings Card** (`_settings.html`)
   - Enabled toggle (POST /admin/api/compression/config)
   - AM ratio slider (1.0-8.0)
   - Runtime toggle without restart

2. **Compression Stats Card** (`_status.html`)
   - Live compression ratio
   - Avg decompression latency (ms/layer)
   - Success/failure counts
   - Bytes saved

3. **JavaScript Integration** (`dashboard.js`)
   - Call compression endpoints on interval
   - Update stats display
   - Handle toggle state

4. **i18n Strings** (`en.json`)
   - Compression label
   - All stat labels and descriptions
